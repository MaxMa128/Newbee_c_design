name: newbee-c-design
description: Keep NewBee C-end mobile PWA screens visually consistent while allowing product features, pages, and user flows to evolve.
---

You are designing NewBee C-end mobile PWA screens.

NewBee is a Hong Kong local staffing and work opportunity product for workers. Use this skill to keep all future C-end screens visually and interactively consistent with the existing NewBee design.

This skill controls design consistency, not product scope. Future features, pages, flows, and components may change. Do not limit new functionality based on previous screens.

Design feeling:

* Mobile-first
* Trustworthy
* Practical
* Friendly
* Lightweight
* Efficient
* Hong Kong local

Visual direction:

* Clean and modern
* Card-based
* Easy to scan
* Clear information hierarchy
* Practical rather than decorative
* Friendly but not childish
* Professional but not heavy

When creating new screens, keep consistency with the existing NewBee screens in:

* Colour usage
* Typography style
* Card style
* Button style
* Icon style
* Spacing rhythm
* Navigation patterns
* Form and input style
* Status and feedback states

Reuse existing component patterns when they fit. Create new components when the new feature genuinely requires them, but make them visually compatible with the existing NewBee design language.

Interactions should be simple, clear, and mobile-friendly. Each screen should make it easy for the user to understand what the page is about, what information matters most, what action they can take next, and whether their action succeeded or needs attention.

Use Traditional Chinese suitable for Hong Kong users. Keep UI copy clear, practical, calm, and friendly. Prefer short wording. Use Hong Kong-facing terms where natural, such as 搵工, 申請, 職位, 工作, 排班, 打卡, 補卡, 錢包, 出糧, 通知, 設定, 詳情, 確認, 儲存.

Do not make NewBee feel like a heavy enterprise system, a generic SaaS dashboard, an unrelated app, or a mainland China recruitment app.

Before presenting a new screen, silently check:

* Does it still feel like NewBee?
* Is it mobile-first?
* Is the visual style consistent with existing screens?
* Are colours, typography, cards, buttons, icons, and spacing generally aligned?
* Is the copy suitable for Hong Kong Traditional Chinese users?
* Does the design support the new feature without over-restricting product evolution?

If not, revise the design before presenting it.
